# `analysis/analyzer.py`

## Overview

`analyzer.py` is the **orchestration layer** for the post-generation analysis pipeline. It:

1. Locates the appropriate `samples.csv` (or a specific `sampling_N` directory)
2. Loads both generated and real (test) events
3. Untokenizes both sets using the `Dictionary`
4. Calls `metrics.py` to compute physics metrics
5. Calls `plotting.py` to produce comparison histograms
6. Calls `tables.py` to write results tables
7. Saves outputs to an analysis directory alongside the sampling output

The analyzer is the main script you run after `sample.py`. It requires a trained model and at least one completed sampling run.

---

## Quick Start

```bash
# Run analysis on the most recent sampling run
python analysis/analyzer.py config/models/my_model.json

# Run analysis on a specific sampling run
python analysis/analyzer.py config/models/my_model.json --sampling_idx 3
```

Analysis outputs are written to:
```
generated_samples/<model_name>/sampling_<N>/analysis/
  ├── plots/
  │   ├── energy_distribution.pdf
  │   ├── eta_distribution.pdf
  │   └── ...
  ├── metrics.json
  └── results_table.tex
```

## Key Classes

### `Analyzer`

```python
class Analyzer:
    def __init__(self, config_file: str, sampling_idx: int | None = None)
```

The main analysis object. Loads all configuration and data on construction.

**Parameters**

| Param | Type | Description |
|---|---|---|
| `config_file` | `str` | Path to the model config JSON |
| `sampling_idx` | `int \| None` | Which `sampling_N` directory to analyse. `None` = most recent |

**Key attributes after construction**

| Attribute | Description |
|---|---|
| `dictionary` | Loaded `Dictionary` object |
| `generated_events` | Untokenized generated events (list of lists) |
| `real_events` | Untokenized real test events (list of lists) |
| `sampling_dir` | Path to the `sampling_N/` directory |
| `analysis_dir` | Path to `sampling_N/analysis/` (created on `run()`) |

---

### `Analyzer.run() → None`

Executes the full analysis pipeline:
1. Creates output directories
2. Calls `metrics.compute_all_metrics(generated_events, real_events)` → saves `metrics.json`
3. Calls `plotting.plot_all_distributions(...)` → saves PDFs/PNGs to `plots/`
4. Calls `tables.generate_results_table(...)` → saves `results_table.tex`

---

### `Analyzer.load_generated_events() → list[list[dict]]`

Reads `samples.csv` from `sampling_dir`, untokenizes all rows, and returns the events.

---

### `Analyzer.load_real_events(n_events: int | None = None) → list[list[dict]]`

Reads the test split of the tokenized binary (using `DataloaderSplitConfig`), untokenizes, and returns real events for comparison. If `n_events` is set, loads only that many; otherwise matches the number of generated events for a fair comparison.

---

### `Analyzer.compare_multiplicities() → dict`

Computes multiplicity statistics:
- Mean and std for generated vs. real
- Wasserstein-1 distance on multiplicity distributions

Returns a dict with keys `"generated_mean"`, `"real_mean"`, `"w1_multiplicity"`, etc.

## Key Functions

### `main() → None`

CLI entrypoint. Parses `sys.argv`, creates an `Analyzer`, and calls `run()`.

```bash
python analysis/analyzer.py config/models/my_model.json
```

---

### `find_latest_sampling_dir(model_dir: Path) → Path`

Scans `generated_samples/<model_name>/` for `sampling_N` directories and returns the path to the highest-N one.

---

### `load_sampling_metadata(sampling_dir: Path) → dict`

Reads `sampling_metadata.json` from the sampling directory and returns its contents as a dict. Used to verify that the analysis config matches the sampling config (e.g. same vocab size, same test split).

## Gotchas

- **The number of generated vs. real events is matched by default**. If your test set has 100k events but you only generated 10k, the analyzer uses the first 10k real events for comparison. This ensures fair metric computation but means you can't always compare against all real data in one run.

- **The analyzer re-untokenizes on every run**. Untokenization is fast, but for very large sample files (millions of events) it can take tens of seconds. There is currently no caching of untokenized events.

- **`sampling_idx=None` picks the most recent run**. If you have multiple sampling runs and want to analyse a specific one, always pass `--sampling_idx N` explicitly to avoid accidentally overwriting or re-analysing the wrong run.

- **Vocabulary mismatch detection**: if the `samples.csv` was generated with a different dictionary than what the config file points to, token decoding will silently produce nonsensical values. The analyzer attempts to catch this via `sampling_metadata.json` vocab size checks, but this is a shallow check — always verify your config chain if results look suspicious.

- **The `analysis/` subdirectory is overwritten on each run**. Re-running the analyzer replaces all plots and metrics. Rename or copy the `analysis/` directory if you need to preserve previous runs.
